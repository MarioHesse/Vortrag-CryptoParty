4. Zwei-Faktor-Authentifizierung

— SMS —

— OTP —

— FIDO2 —

— Biometrie —

Authentifizierung

  • Besitz
    • Sicherheitskarte, Schlüssel, …
  • Wissen
    • Passwort, PIN, TAN, ...
  • Biometrie
    • Fingerabdruck, Iris, Stimme, Gang, ...

— SMS —

  • wird bei Bedarf zugestellt
  • kann nur ein Mal verwendet werden

— SMS —

    Vorteil:
    • Geheimnis wird über separaten Kanal zugestellt
    • minimaler Aufwand für Anwender*in

— SMS —

    Nachteil:
    • Geheimnis wird als Klartext übertragen
    • anfällig, wenn beide Kanäle auf einem Gerät

— OTP —

One-time password

  • kann nur ein Mal verwendet werden
  • Typen
    • Liste (TAN Liste der Bank)
    • Generator (ChipTAN, FreeOTP)

— OTP: Liste —

One-time password

  • vorgefertigte Listen auf beiden Seiten
  • Geheimnisse werden bei Bedarf übertragen

— OTP: Liste —

One-time password

    Nachteile:
    • Verlust der Liste
    • dumm, wenn Liste in Hände der Angreifer fällt

— OTP: Generator —

One-time password

  • Generator erzeugt automatisch ein Passwort
  • 3 Verfahren:
    • Zeitgesteuert (1 - 15 min.)
    • Ereignisgesteuert
    • Challenge-Response-gesteuert

— OTP: Generator —

One-time password

    Vorteile:
    • Passwort wird nur erzeugt, wenn es gebraucht wird
    • Geheimnis muss nur in eine Richtung übertragen werden
    • mit Smartphone relativ bequem

— FIDO2 —

Fast IDentity Online
  • basiert auf asynchroner Verschlüsselung (ähnlich wie PGP)
  • Login ohne Passwort möglich
  • Sicherheit durch PIN am FIDO2 Gerät

— Biometrie —

Individuelle Körpermerkmale

    Vorteil:
    • lässt sich eindeutig zuordnen
    • kann nicht verloren gehen
    • ist nicht ohne Weiteres nachzustellen

— Biometrie —

Individuelle Körpermerkmale

    Nachteil:
    • hinterlassen von Spuren
    • kann nicht gewechselt werden
    • kann u.U. fehlerhaft sein
    • Preisgabe weitere Informationen (Geschlecht, Gesundheit, Ethnie, …)
    • hohe Überwachungsgefahr

Best Practice im Cyberspace

Allgemein

  • abwesend?
      • → Rechner sperren
  • bleib auf dem neusten Stand
      • → Updates immer installieren
  • wöchentliche Backups
      • → kein Backup - kein Mitleid

Browser

  • Passwörter nicht im Browser speichern
      • → Passwort Manager
  • einloggen
      • → ausloggen
  • weniger ist mehr
      • → keine 1000 Add-ons

Passwort

  • Ein Account → ein Passwort
  • Passwort Eingabe
      • → Dienst überprüfen (URL, HTTPS, Zertifikat)
  • Passwort kompromittiert
      • → wechseln
  • Default Passwort
      • → sofort wechseln
  • Schwaches Passwort
      • → sofort wechseln

E-Mail

  • HTML Inhalte → nicht nachladen
  • Links → nicht klicken
  • Absender → kann gefälscht sein
  • Ist eine Postkarte → mit PGP ein Brief

Cloud

  • Es gibt keine Cloud, nur anderer Leute Computer.

Anderer Leute Computer
  • überlege, was du welchem Dienst anvertraust (Land, Rechtsprechung, Interesse, …)
  • Diensten, die du nutzt, musst du vertrauen
  • besser ist es, das nicht zu müssen
      • → Daten verschlüsseln
